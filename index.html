Technical Pipeline

The 5-Stage Parasolid Architecture

1
File Ingestion
Convert 2D/3D drawing from DWG, DXF, STEP, IGES into neutral B-Rep memory.
2
Geometry Kernel
Parasolid engine (OCCT) computes surface, volumes, tolerances, etc.).
3
Feature Extraction
ResNet ML and rule classifiers specific features (e.g. holes, threads, cylinder, etc.)
4
Decision Engine
Via rule and LLM reasoning for a structured-production process sequence.
5
Quotation Generation
Formatting JSON into BoQ and procurement plans.

Deterministic Engineering Data

We combine geometry-based CAD parsing with manufacturing rules, ensuring that dimensions and features are extracted from engineering data to avoid inconcistencies.

A detected hole, thread, or feature are linked to relevant manufacturing processes for areliable foundation for quotation and procurement preparation.

SME Drawing Fallbacks

We solve unstructured drawing issues by Python-OCC and LibreCAD SDK to validate and structure boundary geometry before execution.

Then, we build a library of Parasolid models based on historical part types to enhance processing uptime.
