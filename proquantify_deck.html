Technical Pipeline

The 5-Stage Parasolid Architecture

1
File Ingestion
Parses drawing layers and GD&T from DWG, DXF, STEP into a neutral B-Rep solid memory model.
2
Geometry Kernel
OCCT / Parasolid engine maps exact boundary geometry (volumes, surface areas, wall thickness).
3
Feature Extraction
ResNet ML & rule classifiers interrogate B-Rep to detect specific features (holes, threads, Ra values).
4
Decision Engine
Maps sequences via rule/LLM layers, multiplying by material & localized setup costs.
5
BoQ Assembly
Claude API formats structured JSON into client-ready PDF BoQs and Excel procurement plans.

Deterministic, Not Guessed.

Standard AI forces engineers to guess if a shaft is 32mm or 35mm from a flat image. We use the Parasolid path. The geometry is the ground truth.

A detected blind hole of 8mm diameter at 20mm depth with an M8 thread maps directly to a CNC tapping cycle with known tool and setup costs—creating a traceable, defensible quote.

SME Drawing Fallbacks

SME CAD files are not always well-structured. We solve this natively at Stage 1, using Python-OCC and LibreCAD SDK to validate and structure boundary representations.

Moving forward, we are building a proprietary fallback library of Parasolid models based on specific historical part types to guarantee 100% processing uptime.
